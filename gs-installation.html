Installation

Learn how to install your own Cantiga instance

Requirements

Minimum server requirements:

  • Any HTTP server
  • PHP 7.0
  • The following PHP extensions: gd, intl, calendar, PDO, openssl
  • MySQL or MariaDB database
  • Any mail server

Recommended additional features:

  • Cron or any other task scheduler
  • Shell access to the server

Installation

Visit Download page and get a tarball with the sources of Cantiga, or clone them directly from Git repository:

# mkdir cantiga
# cd cantiga
# git clone https://github.com/zyxist/cantiga.git .

Create an empty MySQL/MariaDB database and install the SQL files from the db/schema directory in the order they are enumerated. Finally, install the SQL file db/fixtures/initial.sql. You can also change the login of the default user, by editing the only row in cantiga_users table.

The final step is editing some files. Create a new file app/config/parameters.yml and copy the contents of app/config/parameters.yml.dist to it. Explanation of options:

  • database_host - host the database is available at,
  • database_port - database port (leave null to use default one),
  • database_name - database name,
  • database_user - db user with rights to access that database,
  • database_password - db password,

The options starting with mailer specify the settings of the mail server. If you install the system on a production server, you must provide the configuration of the production SMTP server and account used for sending the e-mails. For development purposes, you can use the mailcatcher application which emulates a mail server and provides a nice web interface to inspect the mailing.

Remaining options:

  • secret - put here a random string and do not share it with anyone,
  • locale - default language,
  • defaultTimezone - default timezone, specified as Country/City,
  • sourceMail - mail sender address added to the sent mail messages,
  • recaptcha_siteKey and recaptcha_secretKey - integration with reCaptcha.

Create the app/Branding.php and copy the contents of app/Branding.example.php to it. Here you can customize the application, add a custom name, logo, etc.

That's all. If you have a standalone web server, point it to web/ directory as the home directory. Do not make the rest of the directories accessible through the web server. For development purposes, you can use a HTTP server embedded in PHP. Just run the following command:

# php bin/console server:run

The web application will be available under http://localhost:8000/ address. The default credentials are administrator - Admin56789
